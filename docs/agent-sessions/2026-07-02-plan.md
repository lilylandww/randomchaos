# Plan: Fix Code Review Issues

## Summary
Apply four targeted fixes found during code review of the Random Chaos mod.

## Fixes

### Fix 1 — ChaosLifecycle.java — Adventure mode restore logic
**Problem**: Conditional block only restored game mode when event was still active. Players offline during expiry got stuck in adventure mode forever.
**Solution**: Replace conditional block (lines 29-38) with unconditional `restoreIfSaved()` call via direct registry lookup. Remove unused `Identifier` import.

### Fix 2 — PhoenixPathEvent.java — Directional sturdiness check
**Problem**: `isSolidRender()` requires all 6 faces solid — top slabs/stairs return false.
**Solution**: Replace with `isFaceSturdy(level, pos, Direction.UP)` to check only the top face.

### Fix 3 — LeavesOnBreakEvent.java — Persistent leaves
**Problem**: `defaultBlockState()` has `persistent=false`, causing underground leaves to decay on random tick.
**Solution**: Set `LeavesBlock.PERSISTENT = true` on placed leaves.

### Fix 4 — ClayFillEvent.java — Inventory slot restriction
**Problem**: `getContainerSize()` returns 41, filling armor (36-39) and offhand (40) with dead clay items.
**Solution**: Loop only over slots 0-35 (hotbar + main inventory).
