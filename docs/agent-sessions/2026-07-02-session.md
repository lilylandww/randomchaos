# 2026-07-02 Session: Fix ChaosStatePayload networking crash

## Goal
Fix server crash when a player connected with `ExceptionInInitializerError` / `ClassCastException` to `DiscardedPayload`.

## Root Causes
1. `ChaosStatePayload.java:22`: `CustomPacketPayload.createType(id.toString())` applied default namespace, producing invalid `minecraft:randomchaos:state` (two colons).
2. `ChaosStatePayload` S2C stream codec was never registered with `PayloadTypeRegistry`, so the networking layer fell back to the discard codec, causing `ClassCastException` during encoding.

## Changes
- `src/main/java/org/tupi/randomchaos/net/ChaosStatePayload.java` — use `new CustomPacketPayload.Type<>(Identifier)` instead of `createType(toString())`
- `src/main/java/org/tupi/randomchaos/RandomChaosMod.java` — register S2C codec via `PayloadTypeRegistry.clientboundPlay().register(TYPE, STREAM_CODEC)`
- `.gitignore` — scope `net/` rule to top-level only (`/net/`) so `src/.../net/` package is tracked
